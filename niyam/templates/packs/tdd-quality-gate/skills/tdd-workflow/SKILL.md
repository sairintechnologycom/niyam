# Test-Driven Development Workflow

Maintain a strict Red-Green-Refactor feedback loop for all feature implementation:

1. **RED:** Write clear, failing tests targeting the acceptance criteria of the task contract. Ensure validation commands verify the test fails.
2. **GREEN:** Write the minimal implementation code to make all test cases pass.
3. **REFACTOR:** Clean up duplication, formatting, and naming choices while maintaining passing test status.
