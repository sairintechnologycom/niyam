# Implementation Planning

Before writing code, create a clear implementation plan.

## When to Use

- Before implementing any feature or change
- After requirements are clarified
- When the scope involves multiple files or components

## Process

1. **Understand the requirement** — read it carefully, ask clarifying questions
2. **Identify affected files** — list every file that needs to change
3. **Define the approach** — describe what each change does and why
4. **Identify risks** — what could go wrong? what are the edge cases?
5. **Define acceptance criteria** — how will you know it's done?
6. **Estimate scope** — is this a small, medium, or large change?

## Output

Create a plan with:
- Goal (one sentence)
- Files to modify (with paths)
- Implementation steps (ordered)
- Tests to write
- Acceptance criteria
- Open questions (if any)

## Rules

- Do NOT start coding before the plan is reviewed
- Keep plans concise — bullet points, not essays
- Include exact file paths, not vague descriptions
